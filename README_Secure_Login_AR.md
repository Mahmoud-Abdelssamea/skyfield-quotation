# تشغيل نظام Login / Register الآمن

## الملفات

- `SkyField_Management_UI_v8_secure_auth.html`: البرنامج بعد إضافة تسجيل الدخول وإنشاء الحساب.
- `Supabase_Auth_RLS_Setup.sql`: إنشاء جدول البيانات الخاص بكل مستخدم وسياسات الحماية.

## 1. تنفيذ قاعدة البيانات

افتح Supabase ثم:

`SQL Editor → New query`

الصق محتوى ملف `Supabase_Auth_RLS_Setup.sql` واضغط **Run**. يجب أن تظهر نتيجة نجاح بدون أخطاء.

البرنامج الجديد يستخدم جدولًا خاصًا باسم:

`sf_user_store`

كل صف يحتوي على `user_id`، لذلك لا يستطيع أي حساب قراءة أو تعديل بيانات حساب آخر.

## 2. إعداد تسجيل الحساب بالبريد

من Supabase افتح:

`Authentication → Providers → Email`

تأكد أن Email Provider مفعّل. يفضّل إبقاء **Confirm email** مفعّلًا عند الاستخدام الفعلي.

ثم افتح:

`Authentication → URL Configuration`

ضع رابط الموقع المنشور داخل **Site URL** وداخل **Redirect URLs**. مثال GitHub Pages:

`https://YOUR-GITHUB-NAME.github.io/YOUR-REPOSITORY/`

يمكن إضافة رابط الاختبار المحلي أيضًا، مثل:

`http://127.0.0.1:5500/**`

## 3. تشغيل البرنامج

لا تفتح الملف بطريقة `file://` عند اختبار تأكيد البريد أو استعادة كلمة المرور. شغّله من VS Code باستخدام **Live Server** أو انشره على GitHub Pages.

## 4. إنشاء أول حساب

1. افتح البرنامج.
2. اختر **Register**.
3. أدخل الاسم والبريد وكلمة مرور لا تقل عن 8 أحرف.
4. افتح رسالة Supabase واضغط رابط تأكيد البريد، إذا كان التأكيد مفعّلًا.
5. ارجع إلى البرنامج وسجّل الدخول.

## 5. نقل البيانات القديمة

عند أول تسجيل دخول، إذا كان حسابك السحابي فارغًا وكانت البيانات القديمة ما تزال محفوظة في نفس المتصفح، سيقوم البرنامج برفعها تلقائيًا إلى حسابك الجديد.

احتفظ بنسخة JSON احتياطية قبل التجربة.

إذا كانت البيانات القديمة موجودة فقط داخل جدول `sf_kv_store` في Supabase، استخدم جزء **OPTIONAL migration** الموجود في نهاية ملف SQL بعد استبدال `YOUR_USER_UUID` بمعرّف حسابك من `Authentication → Users`.

## 6. الحماية المطبقة

- لا يمكن فتح البرنامج بدون جلسة دخول صحيحة.
- الجلسة تُحفظ وتُجدد تلقائيًا بواسطة Supabase Auth.
- كل مستخدم يرى بياناته فقط بواسطة RLS و`auth.uid()`.
- عند تسجيل الخروج، يتم حذف نسخة بيانات المستخدم الحساسة من المتصفح.
- لا يوجد `service_role` أو Secret Key داخل ملف HTML.
- استعادة كلمة المرور متاحة من شاشة Login.

## ملاحظة مهمة

هذا الإصدار يعزل البيانات حسب الحساب. إذا أردت لاحقًا أن يعمل عدة موظفين على **نفس بيانات الشركة** مع صلاحيات مثل Admin / Accountant / Engineer / Viewer، نحتاج إضافة نظام فرق ومستخدمين وأدوار بدل العزل الفردي الحالي.
